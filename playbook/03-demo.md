# 03 · 데모(피치) 플레이북 — 만든 걸 'S-tier'로 보여주기

> **빌드만큼 데모가 점수다.** 해커톤 본선/결선은 당일 *라이브 데모*로 평가된다.
> 잘 만들고도 못 보여주면 진다. (근거: PostHog "How to demo" — 아래 24팁을 우리 맥락으로 적응)
> **핵심 절약:** 데모 스크립트를 새로 만들지 마라. **`02-project-template` §5 핵심가치 + §10 검증 시나리오**를 그대로 데모로 쓴다.

## 0. 한 줄 원칙
**하나의 핵심 메시지 + 가장 빠른 가치 도달 + 능동적 라이브 시연 + 즉시 행동 마무리.**

## 1. 구조 — '제품 투어'가 아니라 '피치'
- **기억시킬 핵심 메시지 1개**를 정하고 데모의 모든 요소를 그 주위로 배치. (= `PROJECT.md` §5 Core Value)
- **핵심으로 즉시 진입.** 아이디어 배경·서론 생략. (= 출력 계약 "10초 안에 가치" · TTFV)
- 기능 하나씩 둘러보기 ❌ → **문제→해결 한 방의 피치** ⭕.
- **즉시 행동 가능한 마무리로 끝.** (= Rule④ 결과물 + "다음 행동")

## 2. 스토리텔링 (우리 맥락으로)
- **청중 공통의 불편으로 시작.** (= `PROJECT.md` §1 Problem — 예: "내가 뭘 받을 수 있는지 모름" / "민원 어디 넣을지 모름")
- **익숙한 개념을 빌려** 새 개념 설명.
- **"당신이 …" 화법**으로 심사위원을 *사용자 관점*에 앉힌다.
- **대안과 비교 시연** ("원래는 흩어진 검색을 일일이 → 이젠 한 번에").
- **작동 원리·아키텍처는 뒤로.** 질문 받으면 그때.

## 3. 셋업 체크리스트 — 사고 예방 (이걸로 데모가 산다/죽는다)
- [ ] **데모 전용 프로젝트/데이터**로 (실험 흔적 없는 깨끗한 상태)
- [ ] **알림 OFF** — 노트북 알림 끄고 휴대폰 무음
- [ ] **데모 URL 북마크** / Streamlit 포트 고정(`--server.port` 동일)
- [ ] **WiFi/LLM 장애 대비책** — *우리 강점*: 앱이 **키 없이도 mock·폴백으로 동작**하게 설계(→ `stack/`). 최후엔 화면 녹화 백업.
- [ ] **브라우저 125~150% 확대** — 뒷자리 가독성
- [ ] **시작 전 프로젝터/화면공유 테스트**

## 4. 빠르고 안정적으로 보이기 (빌드 결정과 직결)
- **실데이터 사용** — 그럴듯한 가짜 ❌. (= `data-patterns/` 고정 데이터 + 출처)
- **사전 로드·캐싱** — 무거운 호출은 미리. (Streamlit `st.cache_data` / 결과 프리컴퓨트)
- **빈 화면 금지·예시 프리필** — 진입 즉시 가치가 보이게. (= `lessons/` empty state)
- **최소 1회 소리 내어 리허설.** 시간(60~90초) 재기.
- **완벽주의가 데모를 막지 않게.** 좁은 happy-path 1개를 끝까지 완주(= `lessons/mvp.md`). 안 되는 기능은 안 보여주면 그만.

## 5. 지루하지 않게
- **평범한 코드/슬라이드만 띄우기 ❌ → 능동적 라이브 데모가 항상 승리.**
- **확대·하이라이트·애니메이션**으로 시선 유도(Screen Studio 등).
- 시각자료가 *예쁠* 필요는 없다(작동이 우선). 단 **디자인 토큰 기본값**(→ `design/`)으로 깔끔함은 공짜로 챙긴다.
- **내레이션/오디오 과소평가 금지.** 에너지 있게, **사과 금지**("아직 미완성이라…" ❌), **끝났음을 명확히**.
- **"더 기이하게(Do more weird)"** — 기억에 남는 한 방 하나.

## 데모 스크립트 템플릿 (60~90초)
1. **(공통 불편)** "…해보신 적 있죠? 보통 여기서 포기합니다."
2. **(핵심 메시지)** "○○만 하면 ○○를 한 번에."
3. **(라이브)** 입력 → **10초 안에 결과** → **산출물(다운로드·문서·체크리스트)** 까지 *실제로* 보여줌.
4. **(대비)** "원래는 …, 이젠 …."
5. **(마무리·행동)** "지금 ○○만 하시면 됩니다." — 깔끔하게 끝.

> 스크립트 = **검증 시나리오(§10) 재사용**. 통과시킨 그 happy path를, 위 5단계 내레이션을 입혀 보여주면 끝.
