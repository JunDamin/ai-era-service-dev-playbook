# 01 · 기획 인터뷰 (함께 기획 · 상호작용)

> **목적:** 바로 코딩하지 말고, AI가 사용자와 **대화로 기획을 좁힌다.** 결과는 `02-project-template.md`를 채운 기획서 초안 + 해피패스.
> **순서가 핵심:** 문제를 *진짜 문제로 확정*하기 전엔 솔루션으로 넘어가지 않는다. **문제정의가 평가의 심장**(본선 25점 · "문제로 겨룬다").

## AI 진행 규칙 (Claude가 따른다)
- **질문은 하나씩**, 답을 기다린 뒤 다음으로. (한 번에 쏟지 말 것)
- 각 질문에 **추천답(1안)을 먼저 제시** → 사용자는 고르거나 고친다. (빈 화면 금지 = 인터뷰에도 적용)
- 답이 모이면 즉시 `02-project-template`의 해당 칸을 채워 보여준다.
- 모호하면 **구체 시나리오/예시로 되묻는다.** 범위가 커지면 "그건 non-goals(v2)로 둘까요?"라고 막는다.
- **A단계(문제)에선 솔루션 금지** — 기능·화면 얘기가 나오면 "그건 B단계에서, 지금은 문제만 보죠"로 막는다.

---

## A단계 · 문제 정의 (먼저 · 솔루션 금지) ★ 평가의 심장
> 도구: **5 Whys**(깊이) · **5W1H**(커버리지) · fishbone(옵션). 단 *목표는 다이어그램이 아니라* **"구체적 국민 1인 + 진짜 실패 지점 한 문장 + 왜 중요한가"**. 도구가 산출물이 되면 진다.

1. **WHO — 누구의 문제인가?** 구체적 1인(이름·상황·디지털 친숙도). 막연한 '국민' ❌ → "○○동 독거 어르신 김씨". → §2 대상
2. **WHAT(표면) — 그가 *말하는* 문제는?** 그의 언어로 한 문장. (아직 진짜 문제 아님 — 표면) → §1 표면 문제
3. **NOW — 지금은 어떻게 하나?** 현재 흐름·도구·걸리는 시간, 어디서(WHERE)·언제(WHEN) 겪나. → §3 현재 흐름
4. **★ WHY×N — 표면을 파서 *진짜 실패 지점*까지 (5 Whys).** "왜 거기서 막히죠?"를 반복(보통 2~3회면 닿음).
   - **가드레일: 4시간 앱이 손댈 수 있는 실패에서 멈춘다** (예산·정치 등 추상으로 새지 말 것).
   - 결과 = *진짜 문제* 한 문장: **"표면은 X처럼 보이나, 실제로는 Y에서 실패한다."** → §1 진짜 문제 · §4 Pain
   - (대회 출제형이면 `본선_기본전략` §10 '숨겨진 문제 찾기'와 동일: 출제 → 말하는 문제 → 실제 실패 → 해결)
5. **커버리지 점검 (5W1H 한 줄)** — WHO·**WHERE(지역성 "우리 동네")**·WHEN에 빠진 게 없나 빠르게 훑는다. *원인이 진짜 여러 갈래면* fishbone(사람·절차·데이터·환경)으로 한 번 분류 — 보통은 생략.
6. **WHY-MATTERS — 안 풀면?** 누구에게 무슨 일이, 규모는(국민 체감). **가능하면 공공통계로 근거화** — 인구분포·연도별 추이·지역 밀도(KOSIS·통계지리 SGIS·공공데이터포털). 규모·지역성을 *주장이 아니라 숫자*로. (현황 대시보드를 미리 만들어두면 여기 바로 투입 → `stack/templates/` 대시보드 블록) → §1 영향 = **국민 혜택(40점)·문제정의(25점)**로 직결.

> **A단계 종료 게이트:** "구체적 1인 + 진짜 실패 한 문장 + 왜 중요한가"가 손에 있나? **없으면 B로 넘어가지 마라.**

## B단계 · 해결 설계 (문제 확정 후에만)
7. **가장 작은 가치 단위는?** "[대상]이 [과업]을 [결과물]로 끝낸다." → §5 핵심 가치 · `lessons/mvp.md`
8. **★ UX 철학 — 사용자는 어떤 상태로 도착하나?** (감정·디지털 친숙도: 불안한 임차인? 잠 못 잔 보호자?) **첫 화면 10초에 '무엇 하나'를 이해시키나?** 그 단순함을 위해 **무엇을 숨기거나 대신해 줄까?** → §5 서비스 철학
   - *왜 묻나:* 성공한 서비스는 기능이 아니라 철학으로 갈린다(토스 Simplicity — "1초 만에 이해, 배울 것 없음"). 이 답이 입력 패턴·라이팅 톤·화면 구성을 전부 결정한다. **건너뛰지 말 것.**
9. **절대 만들지 않을 것은?** (로그인·권한·다국어·결제 기본 제외) → §6 MVP/non-goals
10. **입력·출력은?** 선택>입력 / 결과물 형태 → §7 입출력
11. **틀리면·비면·느리면?** 실패 시나리오 → §9 실패
12. **성공 기준(끝)은?** pass/fail 1개 → §10 검증

## 마무리 (인터뷰 종료 시)
- §1~§10 + **해피패스 유저저니 5칸**(도착 → … → 완주)을 **기획서 초안**으로 요약 제시 → 사용자 승인. (이 5칸이 사람↔AI 공용 기준 = 정렬·검증·발표·데모 겸용) → §5b
- **와이어프레임·정보구조도(IA)·플로우차트는 안 그린다** — 패턴(A/B/C/D)+템플릿+`tokens.css`가 기본값, AI가 진짜 화면으로 대체.
- 승인되면 **§11 AI Coding Instructions**(개발 지시서)를 생성하고, 문제를 `patterns/README.md`의 모듈(A/B/C/D)에 매핑해 빌드로 넘어간다.
- 빌드 전 체크: `lessons/README.md` 항상-적용 체크리스트.

## 한 줄
> "문제를 *진짜 문제*로 좁히기 전엔 코딩하지 않는다. 5 Whys로 표면을 파고, 5W1H로 빠짐을 막고, 해피패스로 정렬한다."
