# 01 · 기획 인터뷰 (함께 기획 · 상호작용)

> **목적:** 바로 코딩하지 말고, AI가 사용자와 **대화로 기획을 좁힌다.** 결과는 `02-project-template.md`를 채운 기획서 초안.

## AI 진행 규칙 (Claude가 따른다)
- **질문은 하나씩**, 답을 기다린 뒤 다음으로. (한 번에 쏟지 말 것)
- 각 질문에 **추천답(1안)을 먼저 제시** → 사용자는 고르거나 고친다. (빈 화면 금지 = 인터뷰에도 적용)
- 답이 모이면 즉시 `02-project-template`의 해당 칸을 채워 보여준다.
- 모호하면 **구체 시나리오/예시로 되묻는다.** 범위가 커지면 "그건 non-goals(v2)로 둘까요?"라고 막는다.

## 질문 순서 (각 답 → 템플릿 칸)
1. **누구의 문제인가?** 구체적 1인(이름·상황) → §2 대상
2. **무엇이 문제인가?** 한 문장 → §1 문제
3. **지금은 어떻게 하나?** 현재 흐름·시간 → §3 현재 흐름
4. **어디서 막히나?** 가장 아픈 1개 → §4 Pain
5. **가장 작은 가치 단위는?** "[대상]이 [과업]을 [결과물]로 끝낸다" → §5 핵심 가치
6. **절대 만들지 않을 것은?** (로그인·권한·다국어·결제 기본 제외) → §6 MVP/non-goals
7. **입력·출력은?** 선택>입력 / 결과물 형태 → §7 입출력
8. **틀리면·비면·느리면?** 실패 시나리오 → §9 실패
9. **성공 기준(끝)은?** pass/fail 1개 → §10 검증

## 마무리 (인터뷰 종료 시)
- 채워진 §1~§10을 **기획서 초안**으로 요약 제시 → 사용자 승인.
- 승인되면 **§11 AI Coding Instructions**(개발 지시서)를 생성하고 빌드로 넘어간다.
- 빌드 전 체크: `lessons/README.md` 항상-적용 체크리스트.
