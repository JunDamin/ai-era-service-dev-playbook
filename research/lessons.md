# 교훈 (Lessons) — 빠르게 잘 만드는 7원칙

> 딥리서치(Anthropic 1차 출처 25개 검증)에서 뽑은 실전 교훈. 각 교훈 = **원칙 → 왜 → 어떻게 → 4시간 적용**.

## 큰 교훈
> **모델을 더 똑똑하게 만들려 하지 말고, 모델이 일할 '환경'을 설계하라.**
> 컨텍스트·도구·검증·하네스 — 승부는 *모델 주변을 어떻게 엔지니어링하느냐*다.

---

## 1. 컨텍스트는 가장 비싼 자원
- **왜:** 성능은 "얼마나 넣느냐"가 아니라 "무엇을 넣느냐". 비대하면 진짜 지시를 무시.
- **어떻게:** CLAUDE.md <200줄, "빼면 실수하나? 아니면 삭제." 상세는 링크 위임.
- **4시간 적용:** 시작 시 CLAUDE.md엔 5 Rule·파이프라인·스택만. 문제 세부는 별도 메모로.

## 2. 검증 루프가 단일 최대 레버
- **왜:** Claude는 "돼 보이면" 멈춘다. 체크 없으면 'looks done'이 유일 신호.
- **어떻게:** 만들기 전에 완료 기준(pass/fail) 1개 정의 → "통과까지 반복". UI면 브라우저로 사람처럼 끝까지.
- **4시간 적용:** "○○ 시나리오를 네가 직접 클릭해 완주될 때까지 고쳐라"를 첫 지시로.

## 3. 서브에이전트 = 컨텍스트 방어막
- **왜:** 조사·리뷰를 메인에서 하면 컨텍스트 오염·고갈.
- **어떻게:** 조사·정확성 리뷰·적대 검토는 별도 컨텍스트로, 요약만 받기.
- **4시간 적용:** B가 "혜택·서류·기한 정확성"을 서브에이전트로 병렬 점검, A는 빌드 지속.

## 4. 가장 단순한 해법부터 (에이전트를 기본값으로 X)
- **왜:** 자율성↑ = 비용↑ + 오류 누적↑.
- **어떻게:** 단순 프롬프트→명백히 모자랄 때만 다단계. 자율은 샌드박스·가드레일 안.
- **4시간 적용:** Streamlit 단일파일 + LLM 직접 호출. LangGraph 등 무거운 건 금지.

## 5. 도구·인터페이스(ACI)에 프롬프트만큼 공들여라
- **왜:** 도구 많다고 좋아지지 않음. 사람도 못 고를 도구면 모델도 못 고른다.
- **어떻게:** 소수·고임팩트·비중복으로 통합, 결과는 타깃만 반환(전체 나열 X).
- **4시간 적용:** 데이터 조회를 "전체 목록"이 아니라 "조건에 맞는 후보만" 반환하도록.

## 6. 프롬프트는 '구조'와 '적정 고도'
- **왜:** 구조 없으면 오해. 너무 빡빡(if-else)도, 너무 모호도 실패.
- **어떻게:** XML 태그(`<instructions>`,`<input>`), few-shot 3~5(관련·다양·구조화), 섹션화+최소 정보.
- **4시간 적용:** 각 모듈 프롬프트를 `patterns/`에 미리 XML로 박아두고 그대로 사용.

## 7. 장시간 빌드는 '하네스'로 지탱
- **왜:** 모델은 한 방에 끝내려 하거나 조기 완료 선언. 세션 끊기면 기억 0.
- **어떻게:** ① 요구사항 파일=기준점 ② end-to-end 검증 ③ git 커밋 + 진행 파일.
- **4시간 적용:** 0–30분에 "기능 요구사항 파일"부터 작성, 단계마다 커밋.

---

## 한눈에 (외울 7줄)
1. 컨텍스트를 아껴라 (CLAUDE.md 얇게)
2. 완료 기준을 먼저, 통과까지 반복
3. 무거운 일은 서브에이전트로
4. 단순함부터, 자율은 가드레일 안에서
5. 도구는 적게·정확하게, 결과는 타깃만
6. XML 구조 + few-shot 3~5
7. 요구사항 파일·end-to-end·git으로 길게 버텨라
