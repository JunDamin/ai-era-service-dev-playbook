# demo-prep — 데모(피치) 준비

> 진실원천: `playbook/03-demo.md`. **먼저 읽고 그대로 진행하라.** 핵심: 데모 스크립트를 새로 만들지 말고 **통과시킨 검증 시나리오(§10) + 핵심가치(§5)를 재사용**한다.

## 진행 (요약)
1. 프로젝트의 `PROJECT.md`에서 §5 핵심가치(+서비스 철학)와 §10 검증 시나리오를 꺼낸다.
2. `playbook/03-demo.md`의 5단계 스크립트 템플릿(공통 불편 → 핵심 메시지 → 라이브 → 대비 → 행동 마무리)에 끼워 **60~90초 스크립트**를 만든다.
3. §3 셋업 체크리스트를 실제로 점검한다(데모 데이터·알림 OFF·포트 고정·키 없는 폴백 확인·확대·화면 테스트).
4. 앱을 띄워 스크립트대로 **1회 통주 리허설**(브라우저로 직접 클릭) — 막히는 지점은 고치고 재시도.
5. 마감 직전이면 `design-polish` 스킬로 화면을 한 번 다듬는다.
