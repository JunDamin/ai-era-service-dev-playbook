# 문서 생성 (Document, Type D)

> **한 줄:** 입력을 받아 **민원·신청서·보고서 초안**을 생성한다.

## 언제 쓰나
- 민원 초안, 신청서 작성, 보고서 초안류

## 입력 계약
- 선택 + 짧은 텍스트: 사안 유형 · 대상 기관 · 핵심 사실 몇 개. `session_state` 저장.

## 처리 (파이프라인 내 역할)
- Context → 문서 템플릿 선택 → 사실 채움 → 초안 생성(편집 가능)

## 산출물 (Artifact)
- **문서 초안(편집 가능 텍스트)** + 복사 + 다운로드(.md)
- 조합: **A/B/C의 결과를 문서로** 마무리(예: 추천→신청서)

## Claude 프롬프트 (복붙)
```
<instructions>
주어진 사실로 {문서유형} 초안을 한국어 공문 톤으로 작성.
- 사실에 없는 내용 지어내지 말 것(빈칸은 [______]로 표시).
- 정중·명확·간결.
</instructions>
<doc_type>{{type}}</doc_type>
<facts>{{facts}}</facts>
출력: 문서 본문(머리말·본문·맺음) 마크다운
```

## 데이터
- 문서 템플릿(고정) → `data-patterns/` 또는 인라인 템플릿
