# 체크리스트 (Checklist, Type C)

> **한 줄:** 목표·상황을 받아 **준비물·일정·행동 체크리스트**를 생성한다.

## ★ C형 고유 원칙 (출산 행정 체크리스트 드라이런으로 검증)
### ⓐ 항목에 기한이 있으면 — 정적 문자열이 아니라 **계산된 D-day + 상태색**
- 기한이 있는 도메인(신고·신청·접종 등)이면: 기준일(출생일·계약일·D-day)을 입력받고, 항목엔 **상대 규칙**(`deadline_days`)을 둬 **기한일·D-day·상태(🔴지남/🟠임박/🟢여유)를 코드로 계산**하라. 정렬은 급한 순.
- ❌ 안티패턴: `"when": "D-200"` 같은 정적 문자열 — 사용자 시점과 무관해 행동을 못 만든다.
### ⓑ 전부 한 번에 못 한다 — **진행률 + '지금 할 일 1개'**
- 항목이 5개를 넘으면: 진행률 바(X/N)와 **가장 급한 미완료 1개**를 상단에 크게. 완료 체크 시 다음 항목으로 자동 이동.
### ⓒ 조건 따라 갈리는 항목은 — **동적 노출(해당없음=숨김)**
- 상황 조건(맞벌이·다자녀 등)에 따라 해당 여부가 갈리면: 항목에 `conditions[]`를 두고 **켠 조건과 겹칠 때만 노출**. 분모(진행률)도 함께 변해야 한다.
- ※ 주의 — 같은 "미입력"이라도 **유형마다 처리가 정반대**: A=후보 제외 / B=위험 가산 / C=항목 숨김 → `README.md` 빈칸 비대칭 표.
### ⓓ 입력·완료 상태는 **세션에 저장** — 다시 묻지 않는다 (Rule③)
- 기준일·조건·완료 체크 전부 `session_state`(keyed widget). **완료는 항목 고유 `id`에 묶어라** — 조건 토글로 목록이 바뀌어도(인덱스 변동) 체크가 흐트러지지 않게. `chk_{i}` ❌ → `chk_{id}` ✅.

## 언제 쓰나
- 출산 준비, 재난 대비, 절차 준비 등 "무엇을 언제 준비하나"류

## 입력 계약
- 선택형 우선: 상황 · **기준일**(date, D-day 계산 축) · 조건 체크. `session_state` 저장(ⓓ).

## 처리 (파이프라인 내 역할)
- Context → 조건으로 항목 필터(ⓒ) → **기한일·D-day·상태 계산(코드)** → 급한 순 정렬(ⓐ)

## 산출물 (Artifact)
- **상태색 체크리스트(완료 토글, id 기반)** + 진행률 + **지금 할 일 1개**(ⓑ) + 다운로드(.md)
- 조합: **+ A**(추천 항목) · **+ D**(신청 문서)
