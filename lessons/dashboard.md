# Dashboard — 결과 / 대시보드 표시 교훈

> 근거: NN/g 대시보드·전주의적 처리·시각적 위계(1차), Cleveland & McGill 1984.

## 1. 한눈에(at-a-glance)
- **왜:** 대시보드는 한 페이지에서 인지 노력 없이 빠르게 흡수돼야 함.
- **어떻게:** 핵심 지표를 상단 카드로, 부가정보는 아래로.
- **4시간:** 가장 중요한 숫자 3~4개를 큰 카드로 먼저.

## 2. 전주의적 처리(preattentive) 활용
- **왜:** 색·크기·위치 같은 속성은 즉각 지각됨 → 스캔 가능.
- **어떻게:** 중요한 것만 색/크기로 튀게, 나머지는 중립.
- **4시간:** 경고=빨강, 양호=초록 식의 즉각 신호(시그널 색).

## 3. 정량 데이터는 길이·2D 위치로
- **왜:** 사람은 길이·위치는 정확히, **면적·각도는 부정확**하게 추정(파이·버블 부정확).
- **어떻게:** 막대(길이)·선/산점도(2D 위치). 파이차트 지양.
- **4시간:** 비교는 막대그래프 기본. 파이는 피하기.

## 4. 시각적 위계
- **왜:** 중요도 순으로 눈을 유도.
- **어떻게:** 명도·채도 대비·볼드. 강조는 소수만.
- **4시간:** 한 화면에 '주인공' 1개. 나머지는 약하게.

## 5. 결과는 '가져갈 수 있게'
- **왜:** 사용자는 결과를 복사·저장·공유하려 함(행동으로 연결).
- **어떻게:** **복사 버튼·다운로드(PDF/CSV)**·"다음 행동" 안내.
- **4시간:** 모든 결과 카드에 복사/다운로드 + 다음 단계 버튼.

## 6. 점수 말고 '기여'를 보여라 (위험/스코어형)
- **왜:** 종합 점수·등급만 주면 블랙박스 → 불신. 사용자는 "왜 이 색이지?"를 묻는다.
- **어떻게:** 종합 신호색(주인공) 아래에 **신호별 +기여를 펼침**(✅양호 / ❓미확인 / ⚠️위험 + 점수). 합산식이 눈에 보이게.
- **4시간:** High/Med/Low 큰 카드 1개 + 접이식 "왜 이 위험도인가요?" 기여 목록.
- **근거(드라이런):** 전세 위험 점검에서 "10점=위험높음"만으론 설득이 안 됨 → 신호별 +3/+2/+0을 펼치니 행동으로 연결됨. (B형 원칙 ⓒ → `patterns/decision.md`)
- **짝꿍 — 미확인의 표시:** `모름`을 0/공백이 아니라 **❓ + 위험 가산**으로 명시해야 "확인 안 함=위험"이 보인다(`patterns/decision.md` ⓐ).
