# lessons/ — 설계 교훈 지식베이스

코드는 AI가 다시 써준다. **재사용할 자산은 "어떤 기능이 필요한가 / 어떤 실수를 피하나 / 사용자는 어디서 막히나"** 다.
이 폴더는 *남들이 이미 배운 교훈*을 모아 매번 재고민 없이 **선반영**하기 위한 것. (근거: `../research/ux-design-findings.md`)

## ★ 항상 적용 체크리스트 (빌드 시작 전 읽기)
- [ ] **빈 화면 금지** — empty state엔 예시/안내, 빈 스피너 대신 진행상태·스켈레톤
- [ ] **placeholder를 레이블로 쓰지 말 것** — 필드 밖 영구 레이블 + 힌트
- [ ] **선택 > 입력** — 가능한 값은 드롭다운/칩/자동완성(외워 타이핑 X)
- [ ] **오류 예방** — 자유입력보다 제약(피커·선택지), 파괴적 액션엔 확인
- [ ] **검증은 on-blur** — 필드 옆 메시지, 타이핑 중 조기검증 X
- [ ] **좋은 기본값** — 합리적 default로 설정 최소화
- [ ] **결과는 결과물로** — 복사·다운로드 버튼, "다음 행동" 안내
- [ ] **위계** — 가장 중요한 1~2개만 고대비/볼드, 핵심은 앞부분 배치
- [ ] **나중으로** — 로그인·권한·다국어·결제는 MVP에서 제외
