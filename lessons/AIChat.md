# AIChat — AI / 대화형 UX 교훈

> ⚠️ **잠정(보강 필요):** 이 영역은 적대검증 통과 주장이 없어, 아래는 NN/g·실무 1차 출처 기반 *잠정* 정리. 사용 전 재확인 권장.

## 1. 응답시간 = 경험의 핵심
- **왜:** 0.1s 이내=즉각, ~1s=흐름 유지, **10s 넘으면 주의 이탈**(NN/g). Doherty 임계 <400ms면 생산성↑.
- **어떻게:** 즉시 반응(버튼 비활성·로딩), 길면 진행/부분 결과 표시.
- **4시간:** LLM 호출은 길다 → **스트리밍 출력** 또는 단계 표시 필수.

## 2. 빈 스피너 금지 → 진행을 보여줘라
- **왜:** "답변 생성 중"만 보이면 불안·이탈.
- **어떻게:** "이해한 상황 / 발견한 후보 / 현재 분석"을 단계로 노출(스켈레톤·스트리밍).
- **4시간:** 입력 직후 "상황 분석 표시 → 후보 → 결과" 순으로 점진 공개.

## 3. Time-to-First-Value (10초 규칙)
- **왜:** 사용자는 빨리 가치를 느껴야 남는다.
- **어떻게:** 무거운 전체 결과보다 **첫 가치(요약/후보)**를 먼저.
- **4시간:** 완성 결과 대기 X → 부분이라도 즉시 보여주기.

## 4. AI를 '전지적 오라클'로 제시하지 마라
- **왜:** 과신 유발·오류 시 신뢰 붕괴(NN/g 'AI Magic 8-Ball').
- **어떻게:** **Confidence(확정/높음/중간/확인필요)** 표시, **출처 링크**, "왜 이렇게 나왔는지" 설명(explainable).
- **4시간:** 모든 추천에 confidence + 근거/출처. 불확실하면 "확인 필요"로 정직하게.

## 5. 다음 행동을 알려줘라
- **왜:** 정보만 주면 사용자가 막힘(기획 Rule 1: 행동 지원).
- **어떻게:** 결과 끝에 "다음 단계" 버튼/체크리스트.
- **4시간:** 추천 → "지금 신청하기 / 준비서류 보기" 등 행동 CTA.
