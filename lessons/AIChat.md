# AIChat — AI / 대화형 UX 교훈

> 근거: NN/g 1차(응답시간·스켈레톤·환각·설명가능성) — 3차 리서치로 검증. 각 교훈 = 원칙 → 왜 → 어떻게 → 4시간 적용.

## 1. 응답시간 예산
- **왜:** 0.1s=즉각 느낌(피드백 불필요), **10s=주의 유지 한계**(넘으면 진행상태 + 중단 옵션 필수).
- **어떻게:** 짧으면 그냥 결과, 길면 진행/부분결과 표시 + 취소 버튼.
- **4시간:** LLM 호출은 길다 → **스트리밍** 기본, 길어지면 단계 표시 + 중단 가능.

## 2. 빈 스피너 금지 → 스켈레톤/스트리밍
- **왜:** 와이어프레임형 스켈레톤이 *체감 대기*를 줄인다(10s 이하 구간).
- **어떻게:** "답변 생성 중" ❌ → **스켈레톤** 또는 **토큰 스트리밍**(`st.write_stream`)으로 점진 표시.
- **4시간:** 입력 직후 "이해한 상황 → 후보 → 결과" 단계 노출.

## 3. 신뢰 보정 (Trust calibration)
- **왜:** AI를 자신감 있게 제시하면 **부당한 신뢰(과신)**를 부른다.
- **어떻게:**
  - **1인칭 불확실성**("확실치 않지만…")이 일반 경고보다 과신을 더 줄임.
  - **Confidence(High/Med/Low)** 표시 — 단 **보정될 때만**(엉터리 신뢰도 표시는 오히려 오도).
  - 출처를 **눈에 띄게**.
- **4시간:** 추천마다 confidence + 출처 + 불확실하면 "확인 필요" 정직 표기.

## 4. 출처 ≠ 검증 (안티패턴)
- **왜:** AI 출처는 자주 환각(존재X URL)이고, 사용자는 거의 클릭 안 함(~1%) — '진실의 후광' 착시.
- **어떻게:** 출처만으로 안심시키지 말 것. **검증 발판**(고정 데이터 내 생성·재검증·"확인 필요" 라벨)과 함께.
- **4시간:** 생성은 **큐레이션 데이터 안에서만**, 데이터 밖은 지어내지 말고 "확인 필요".

## 5. Time-to-First-Value + 다음 행동
- **왜:** 빨리 가치를 느껴야 남고, 정보만 주면 막힌다(Rule 1: 행동 지원).
- **어떻게:** 첫 가치(요약/후보)를 먼저, 결과 끝엔 "다음 단계" CTA/체크리스트.
- **4시간:** 완성 대기 X → 부분이라도 즉시, 끝에 "지금 신청하기" 등.
