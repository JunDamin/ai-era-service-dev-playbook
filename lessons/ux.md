# UX — 범용 사용성 교훈

> 근거: NN/g 10대 사용성 휴리스틱 등(1차) + **UX 라이팅·에러는 토스(toss.tech) 기준**(현대 한국어 서비스의 사실상 표준 — 해외 가이드와 충돌 시 토스 우선). 각 교훈 = 원칙 → 왜 → 어떻게 → 4시간 적용.

## 1. 시스템 상태 가시성 (휴리스틱 #1)
- **왜:** 사용자는 늘 "지금 무슨 일이 일어나는가"를 알아야 함(합리적 시간 내 피드백).
- **어떻게:** 모든 액션에 즉각 피드백 — 버튼 로딩 상태, 저장됨 토스트, 진행 표시.
- **4시간:** 빈 스피너 ❌ → 스켈레톤/진행 단계 표시를 기본.

## 2. 오류 예방 > 좋은 오류 메시지 (#5)
- **왜:** 최선은 문제가 *발생하지 않게* 하는 것.
- **어떻게:** 좋은 기본값 + 제약(피커·선택지) + 커밋 전 확인.
- **4시간:** 자유 입력 대신 드롭다운/날짜피커, 파괴적 액션엔 확인 단계.

## 3. 인지 > 회상 (#6) — "선택 > 입력"
- **왜:** 외워서 떠올리게 하지 말고 보이게 하라.
- **어떻게:** 가능한 값은 셀렉트/칩/자동완성으로 노출.
- **4시간:** 지역·유형 등은 드롭다운/멀티선택. (상세는 `data-input.md`)

## 4. 심미-사용성 효과
- **왜:** 보기 좋으면 '사소한' 문제는 용서됨 — 단, 큰 결함은 못 가림.
- **어떻게:** 일관된 스페이싱·타이포·색만으로 초기 신뢰 확보.
- **4시간:** 디자인 토큰 1세트(색/간격/폰트)를 먼저 고정하고 시작.

## 5. 시각적 위계
- **왜:** 눈이 중요도 순으로 소비하도록 조직 → 정보 전달을 통제.
- **어떻게:** 명도·채도 대비, 볼드로 신호. **강조는 화면당 소수만.**
- **4시간:** 핵심 숫자/주요 액션 1~2개만 고대비, 나머지는 약하게.

## 6. F-패턴에 '맞서' 설계
- **왜:** 사용자는 상단 가로 스윕 + 좌측 세로로 훑어 큰 덩어리를 놓침.
- **어떻게:** 명확한 헤딩, 볼드 키워드, **핵심을 줄·문단 앞에**, 짧은 단락·불릿.
- **4시간:** 긴 문단 금지. 결론부터, 불릿 위주.

## 7. UX 라이팅 — 화면의 모든 문구에 (토스 8원칙 압축)
- **왜:** 한국 사용자의 문구 기대치는 토스가 설정했다. 공문 톤·전문용어는 그 자체로 이탈 요인.
- **어떻게 (조건 → 처방):**
  - 버튼·링크 문구를 쓸 때 → **다음 화면이 예측되게**("확인" ❌ → "신청서 다운로드" ✅). *Predictable hint*
  - 문장을 다 썼을 때 → **의미 없는 단어·중복 문장 삭제**("앞으로 결제하기" → "결제하기"). *Weed cutting*
  - 안내문이 길어질 때 → **지금 필요한 핵심 메시지 1개만**, 한 문장에 하나. *Focus on key message*
  - 용어를 고를 때 → **소리 내어 말해보고 어색하면 교체**(전문용어 → 일상어). *Easy to speak*
  - 행동을 유도할 때 → **강요·공포 금지**("안 하면 손해" ❌ → 선택권 존중 제안). *Suggest over force* — 특히 위험형(B)에서 경고와 협박의 경계 주의.
  - 톤은 → **해요체 일관** + 능동형 + 긍정형.
- **4시간:** 화면 문구를 마지막에 한 번 훑으며 위 처방으로 잘라내기(보통 30% 줄어든다).

## 8. 에러·실패 메시지 = "다음 행동 안내" (토스 Navigating Error)
- **왜:** 에러의 목적은 사과가 아니라 **사용자를 다음 화면으로 보내는 것.**
- **어떻게:** 3요소 구조 — **무엇이 문제** + **왜** + **어떻게 해결**("시스템 오류" ❌ → "은행 점검 중이에요. 오전 2시 이후 다시 시도해주세요" ✅). 반복 상황은 문구 템플릿화(변수만 교체).
- **4시간:** 우리 보일러플레이트의 LLM 폴백 경고·빈 상태 문구에 그대로 적용 — "실패했습니다" ❌ → "키가 없어 예시 모드로 보여드려요. 실제 결과는 키 설정 후에"식.
