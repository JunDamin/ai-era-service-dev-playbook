# CLAUDE.md 잘 쓰는 법

CLAUDE.md는 **매 세션 컨텍스트에 자동 로드**된다. 비대하면 Claude가 진짜 지시를 무시한다.

## 원칙
- **<200줄** 목표(파일은 길이와 무관하게 전부 로드되지만, 길수록 준수율↓).
- 각 줄에 질문: **"이 줄을 빼면 Claude가 실수하나? 아니면 삭제."**
- **구체적·검증 가능**하게: "코드 잘 포맷해라" ❌ → "2-space 들여쓰기" ✅
- 마크다운 **헤더·불릿으로 구조화**(빽빽한 문단 ❌)

## 포함 (INCLUDE)
- Claude가 추측 못 하는 bash 명령(빌드·실행·테스트 러너)
- 기본값과 다른 코드 스타일 규칙
- 프로젝트 아키텍처 결정 · 레포 관례(브랜치·PR)
- 환경 특이사항(필수 env var) · 비자명한 함정(gotcha)

## 제외 (EXCLUDE)
- 코드에서 추론 가능한 것 · 표준 언어 관례
- 상세 API 문서(→ 링크로 대체) · 자주 바뀌는 정보
- 긴 튜토리얼 · 파일별 설명 · 자명한 모범관행
- 다단계 절차나 한 영역에서만 필요한 맥락 → **skill** 또는 **경로-스코프 규칙**으로 분리

## 한 줄 점검
> "매 세션 항상 필요한 사실만 남긴다 — 빌드 명령, 관례, 레이아웃, '항상 X' 규칙."
