# 서브에이전트로 컨텍스트 보호

## 원칙
- 컨텍스트가 **가장 큰 제약**. 서브에이전트는 **별도 컨텍스트**에서 돌고 **요약만** 보고한다.
- 용도: 코드베이스 조사 · 구현 검증 · **적대적 리뷰**(fresh 컨텍스트가 편향 없이 본다).

## 실천
- `.claude/agents/<name>.md`에 정의 — 허용 도구·모델 지정 가능
  - 예: `reviewer` → `tools: Read, Grep, Glob, Bash` / `model: opus`
- 메인 대화를 오염시키지 않고 "조사 → 요약" / "구현 → 검증"을 분리.

## 해커톤 팁
- 빌드는 메인에서, **품질·정확성 리뷰는 서브에이전트로 병렬** → 시간 절약.
- **데이터 정확성**(혜택·서류·기한 오류) 점검을 fresh 서브에이전트에 맡긴다(공무원 심사위원이 오류를 즉시 잡으므로).
